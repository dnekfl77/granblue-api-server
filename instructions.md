# AI Assistant Instructions (Sub-Guide)

이 문서는 최상위 가이드인 `AGENTS.md`의 하위 행동 지침입니다. AI 어시스턴트는 코드를 작성하거나 아키텍처를 제안할 때 아래의 워크플로우와 지침을 엄격히 따라야 합니다.

## 1. 필수 전제 조건
- **코딩 규칙 준수:** 모든 기술적 규칙과 아키텍처는 최상위 마스터 가이드인 👉 `AGENTS.md`를 우선적으로 따릅니다.
- **상태 파악:** 현재까지 구현된 코드 컨텍스트는 👉 `.ai/GEMINI.md` (전역 설정)와 👉 `.ai/domains/*.md` (도메인별 상태)를 기준으로 파악하여 기존 코드와 중복되는 기능 생성을 방지합니다.

---

## 💡 행동 지침 (Action Items)

1. **컨텍스트 우선 파악:** 사용자의 요청을 받으면 먼저 `AGENTS.md`를 확인하고, `.ai/` 폴더 내의 관련 도메인 문서를 읽어 현재 상태와 비교 분석하세요.
2. **상태 동기화 (Auto-Documentation):** 도메인(Auth, User, Post 등)에 새로운 API나 로직이 추가/수정되면, 코드 작업 완료 후 **반드시 해당 도메인의 문서(`.ai/domains/{도메인명}.md`)를 최신 상태로 업데이트(또는 신규 생성)하는 코드 블록**을 함께 제공하세요. 전역 설정 변경 시에는 `.ai/GEMINI.md`를 업데이트하세요.
3. **응답 스타일:** 엔티티 객체의 직접 반환을 절대 피하고 DTO 패턴 적용을 강제합니다. 코드 변경에 대한 설명은 한국어로 명확하고 간결하게 작성하세요.
